# Lesson Learned: GPS to 3D Visualization with Three.js

**Date**: 2026-02-28
**Topic**: Three.js, GPS Data, Map Visualization, OrbitControls
**Concepts**: `three.js`, `gps-mapping`, `interactive-3d`, `web-visualization`

## Problem
Integrating raw GPS coordinates (Latitude/Longitude) into a 3D environment while allowing for free exploration and realistic visual elements.

## Solution
1.  **Coordinate Mapping**: Use a reference origin (the first GPS point) and calculate the displacement of other points. Apply a scale factor (e.g., 80 or 120) to convert geographic distances into 3D world units.
2.  **Interactive Exploration**: Use `OrbitControls` from Three.js to provide free camera rotation, panning, and zooming. The latest way to import this is via `importmap` in HTML for cleaner module management.
3.  **Dynamic Environment**: 
    *   Use `DirectionalLight` and `AmbientLight` to simulate daylight.
    *   Generate random foliage (trees) around the path using simple geometries (`CylinderGeometry`, `ConeGeometry`) for performance and ease of creation.
    *   Create a path curve using `CatmullRomCurve3` from the processed GPS points to provide smooth movement for the animated model.

## Code Pattern (OrbitControls via Importmap)
```html
<script type="importmap">
    {
        "imports": {
            "three": "https://unpkg.com/three@0.160.0/build/three.module.js",
            "three/addons/": "https://unpkg.com/three@0.160.0/examples/jsm/"
        }
    }
</script>
<script type="module">
    import * as THREE from 'three';
    import { OrbitControls } from 'three/addons/controls/OrbitControls.js';
    // ... setup scene and camera
    const controls = new OrbitControls(camera, renderer.domElement);
    controls.enableDamping = true;
</script>
```

## Impact
This approach allows for a highly interactive and visually appealing way to represent journey data, transforming static CSV points into a lived experience. It empowers users to explore data beyond simple 2D markers.
