## The Problem
When starting a session in a project with a complex or non-standard documentation structure (like the Oracle vault), it's easy to overlook existing contexts or miss established patterns for long-term memory.

## The Insight
The "ψ" (psi) directory (often rendered as "ฯ" in certain environments) acts as a high-signal repository for "nothing deleted" logs, learnings, and retrospectives. Prioritizing this directory's contents (specifically `ψ/memory/learnings`) over a generic exploration of the codebase allows for immediate alignment with the project's intellectual trajectory.

## The Strategy
1. **Always check `ψ/memory/learnings`** at the start of a session.
2. **Consult recent retrospectives** (`ψ/memory/retrospectives/YYYY-MM/DD/`) to understand the previous agent's momentum.
3. **Use the `git log --oneline -10` and `git diff --stat`** to bridge the gap between "what happened in the vault" and "what changed in the code."

## Impact
Aligning with the vault structure ensures that every session contributes to a cohesive, cumulative intelligence rather than isolated, repetitive task execution. This reinforces the "Oracle" identity where the AI is not just a tool, but a growing repository of project wisdom.
