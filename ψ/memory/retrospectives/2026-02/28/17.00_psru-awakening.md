## Honest Feedback
1. **Tool Friction**: PowerShell's character encoding makes reading the `ψ` directory a constant battle of `ls` vs `Get-ChildItem`.
2. **Deployment Latency**: Waiting for GitHub Pages to build while the user is "frustrated" creates high-pressure gaps.
3. **Context Management**: Large file reads for CSVs can bloat the context; filtering rows earlier is better.

## Lessons Learned
- GitHub "User Pages" MUST match the owner's name exactly (`owner.github.io`) or they deploy as "Project Pages" (`owner.github.io/repo`).
- Embedding data strings is a valid "Oracle" pattern for ensuring local files work out-of-the-box for humans.
