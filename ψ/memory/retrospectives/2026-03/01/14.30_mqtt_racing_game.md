## Honest Feedback
1. Attempting to manage third-party tools like Vercel entirely via automated CLI commands is brittle when auth tokens expire or are absent; it breaks context flow entirely. 
2. Creating complex HTML structure dynamically via LLM generation can sometimes neglect mobile-first CSS priorities until explicitly directed.
3. Keeping track of the transition from one GitHub remote URL to an entirely new GitHub account while coordinating the GitHub Pages settings requires careful choreography to avoid ghost 404s.

## Lessons Learned
- When creating simple leaderboards or persistent stats for mini-games, MQTT retained messages (`qos: 1, retain: true`) are an incredibly elegant and maintenance-free alternative to traditional databases.
- The `actions/deploy-pages` step syntax `url: ${{ steps.deployment.outputs.page_url }}` is pedantic and must use dot-notation for its object property expansion.
- Always check the environment capabilities before attempting to run globally installed npm CLI tools (like vercel) that might halt waiting for interactive manual inputs.
