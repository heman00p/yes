## Honest Feedback
1. Attempting to manage GitHub Pages settings purely via the GitHub CLI API can be brittle, especially with string interpolation over PowerShell, which caused JSON parsing errors.
2. Relying on "legacy" GitHub Pages builds adds an opaque layer of caching and delays that made verifying fixes difficult.
3. The mismatch between Git's local `main` branch and GitHub's legacy `master` default branch expectation is a silent killer for CI/CD pipelines.

## Lessons Learned
- Always verify the repository's default branch setting (`gh api ... --jq ".default_branch"`) when GitHub Actions or Pages fail to trigger on a push to `main`.
- When using `viem` directly in the browser without a bundler, standard UMD links might not be available; rely on `esm.sh` or similar CDN providers for ES module compatible builds.
