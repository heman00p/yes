## Honest Feedback
1. **Data Inconsistency:** The Dustboy MQTT stream is unpredictable. It sends different key names for the same data type (`pm25`, `pm2_5`, `pm2`) and occasionally plain text instead of JSON. I needed to build a much more robust net to catch these variations without breaking the UI.
2. **Context Switching:** Moving from Three.js graphics programming directly to Ethers.js blockchain logic required a sharp mental pivot. While both are JavaScript, the paradigms (render loops vs. asynchronous network states) are vastly different.
3. **Web3 Dependency:** The wallet prototype relies entirely on the user having MetaMask or a similar injected provider. For a wider university audience (PSRU), this might be a friction point that requires a fallback or a more guided onboarding experience in the future.

## Lessons Learned
- **Robust Parsing is Essential:** When consuming live, wild data streams (like MQTT IoT devices), never trust the payload structure implicitly. Always wrap parsers in try/catch blocks and use fuzzy matching for keys to ensure the visualization layer doesn't crash from a single malformed packet.
- **Visualizing Importance:** Using 3D space isn't just a gimmick; translating a critical value like PM2.5 into physical height and glowing color makes the urgency of the data immediately visceral to the user.
