## Blockers & Resolutions
- **Blocker**: รถไฟใน 3D Map หายออกไปจากรัศมีกล้อง
  **Resolution**: เข้าไปอัปเดตแกน `camera.position` ให้วิ่งตามกึ่งกลางพิกัดของตัวรถไฟแบบ Frame-by-frame
- **Blocker**: คำสั่ง Terminal บางคำสั่งถูกระบบระงับ (Cancel) ไปเพื่อความปลอดภัยของเครื่องผู้ใช้
  **Resolution**: สลับโหมดแก้ไขปัญหาด้วยการเขียนไฟล์ และให้คู่หู (คุณหมีพูห์) เป็นผู้ตัดสินใจกดดำเนินการด้วยตัวเอง หรือบอกวิธีทำด้วยมือแทน

## Honest Feedback
1. อยากให้มีการอธิบาย Path ระบบ Windows ให้ละเอียดกว่าเดิมใน Repository กลาง (หลายสคริปต์ยังอิง Linux/Mac)
2. เครื่องมือสร้าง 3D บางส่วนยังมีน้ำหนักเยอะ อาจต้องคำนึงถึงประสิทธิภาพ (Performance) หากต้องการเพิ่มจำนวนจุดในอนาคต
3. การส่งคำสั่งผ่าน API ยังค่อนข้างเคร่งครัด (ถูกจับ Cancel ง่าย) ซึ่งเป็นข้อดีเรื่องประหยัดและปลอดภัย แต่บางครั้งก็ขัดจังหวะโฟลว์การทำงานอัตโนมัติ

## Lessons Learned
- **Architecture Constraints Shape Behavior**: ข้อจำกัดของ OS และ Security ก่อให้เกิดโครงสร้างการแก้ปัญหาที่สร้างสรรค์ (เช่น การทำ .bat หรือการพึ่งพามนุษย์)
- **Visuals Dictate Understanding**: ไม่ว่าข้อมูล GPS หรือ PM2.5 จะซับซ้อนแค่ไหน การแปลงมันมาเป็น Data Visulazation เช่น Leaflet หรือ Chart.js ช่วยย่นระยะเวลาทำความเข้าใจของมนุษย์ได้หลายเท่า
- **The Philosophy In Practice**: "Nothing is Deleted" และ "Form and Formless" ทำงานในวันนี้จริงๆ จากโค้ดต้นฉบับ สู่ CV สู่ Dashboard ทั้งหมดเชื่อมโยงเป็นรูปแบบนับไม่ถ้วนที่รับใช้เป้าหมายเดียวกัน
